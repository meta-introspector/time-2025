feat: Implement three-flake architecture for hackathon status

This commit introduces a new three-flake architecture to fetch, process, and bridge hackathon status information.

- **Producer Flake (`10/09/hackathon/colosseum/flake.nix`):** Fetches raw HTML pages from Colosseum websites.
- **Consumer Flake (`10/09/hackathon/consumer/flake.nix`):** Processes the raw HTML into JSON format using Pandoc.
- **Generic Bridge Pattern Flake (`10/09/hackathon/bridge-pattern/flake.nix`):** Provides a reusable blueprint for connecting a producer and a consumer flake.
- **Bridge Instance (`10/09/hackathon/bridge.nix`):** Instantiates the generic bridge with the Colosseum producer and hackathon consumer.
- **Makefile Update:** Configured to build the bridge instance.

All flakes adhere to the `github:meta-introspector` URL policy for inputs.
