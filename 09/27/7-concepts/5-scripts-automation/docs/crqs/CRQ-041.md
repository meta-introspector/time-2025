# CRQ-041: Refactor log-analysis-pipeline Flake Inputs for Module Import

## Title
Refactor `log-analysis-pipeline` Flake Inputs to Explicitly Import Modules from `meta-introspector-flake` Source

## Description
This Change Request addresses a persistent issue encountered during the development of the `log-analysis-pipeline` flake, specifically related to how external Nix modules are imported. The previous approach of directly referencing `.nix` files within a `github:...&dir=...` flake input URL led to errors when Nix attempted to treat these individual `.nix` files as full flakes.

The core problem is that a flake input expects either a directory containing a `flake.nix` or a direct path to a `.nix` file *within* a flake. When a `.nix` file is provided directly as a flake input, Nix attempts to resolve it as a flake, leading to "Not a directory" or "No such file or directory" errors when it looks for a `flake.nix` within that `.nix` file's path.

This CRQ proposes to refactor the `log-analysis-pipeline/flake.nix` to correctly import external Nix modules from the `meta-introspector-flake` (which points to the `time-2025` repository).

## Proposed Changes

1.  **Modify `log-analysis-pipeline/flake.nix` Inputs:**
    *   The `meta-introspector-flake.url` input will point to the root of the `github:meta-introspector/time-2025` repository (e.g., `github:meta-introspector/time-2025?ref=feature/foaf`). This fetches the entire repository as a flake input.
    *   Remove individual flake inputs for `secret-scanner-flake`, `nix2gramIndexerModule`, `nGramGeneratorModule`, and `twoGramReportGeneratorModule`.

2.  **Explicitly Import Modules in `outputs` Section:**
    *   Within the `outputs` section of `log-analysis-pipeline/flake.nix`, the individual `.nix` modules (e.g., `secret_scanner.nix`, `nix_2gram_indexer.nix`, `n_gram_generator.nix`, `2gram_report_generator.nix`) will be explicitly imported using `import "${meta-introspector-flake}/path/to/module.nix" { inherit lib pkgs; };`. This treats them as Nix modules, not as separate flakes.

3.  **Update Internal References:**
    *   All internal references within `log-analysis-pipeline/flake.nix`, `lib/helpers.nix`, and `packages/analysis.nix` that previously used `secret-scanner-flake`, `nix2gramIndexerModule`, `nGramGeneratorModule`, or `twoGramReportGeneratorModule` will be updated to use the newly imported module variables (e.g., `secretScannerModule.scanForSecrets`).

## Rationale
*   **Correct Flake Input Usage:** Adheres to the correct way of importing Nix modules from a repository that is itself a flake input.
*   **Improved Readability:** Centralizes the import logic for external modules.
*   **Maintainability:** Makes it easier to manage dependencies on external Nix modules.
*   **Resolves Build Errors:** Directly addresses the "Not a directory" error when Nix attempts to treat individual `.nix` files as flakes.

## Acceptance Criteria
*   The `log-analysis-pipeline` flake builds successfully without errors related to flake input resolution.
*   All external Nix modules are correctly imported and their functions are accessible.
*   The `make generate-2gram-report` target executes successfully.
*   The `flake.nix` is cleaner and more maintainable.

## Dependencies
*   `github:meta-introspector/time-2025` repository (specifically the `feature/foaf` branch).
*   The existence of the `.nix` module files within the `time-2025` repository at the specified paths (e.g., `2025/10/01/docs/theory/secret_scanner.nix`).
