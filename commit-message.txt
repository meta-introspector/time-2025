feat: Implement ZOS Orchestrator flake

This commit introduces the ZOS Orchestrator flake, which coordinates the Act, Decide, and Observe tasks.

- **Orchestrator Flake (10/15/zos/orchestrator.nix):**
  - Defines inputs for `act`, `decide`, and `observe` flakes.
  - Orchestrates the execution of these flakes in sequence:
    1. Calls `decide` with an initial `orientationDecision`.
    2. Extracts `plan.json` from `decide`'s output.
    3. Calls `act` with the `plan.json` as `actionPlan`.
    4. Extracts `new-state.json` from `act`'s output.
    5. Combines `plan.json` and `new-state.json` into a single JSON string.
    6. Calls `observe` with this combined JSON string as `currentState`.
    7. Exposes the `report.json` from `observe` as its default package output.
  - Includes a basic `checks.healthcheck`.
  - Corrected the exposure of `packages.${system}.default` to properly expose the `observeReport`.

- **Orchestrator Makefile (10/15/zos/Makefile):**
  - Created a `Makefile` with a `test` target for evaluating the orchestrator flake.

This orchestrator completes the Act-Decide-Observe cycle, enabling a continuous feedback loop for system state analysis.
